Git Cheat sheet

Algunos comandos que pueden ayudar cuando colaboren con proyectos muy grandes de github:

$git init (Iniciar repositorio)

$git add "Nombre del archivo"

$git add . (Agregamos todos los archivos que esten en el stage) Nota: Stage se pudiera decir que hay cierta info en la cache de la compu, en el limbo en pocas palabras

$git commit -m "Agregar mensaje del cambio" (Con commit podemos darle un "submit" a los archivos que actualizemos y modifiquemos )

$git commit -am "agrega tu mensaje aqui" (Este commit hace que ya no tengas que agregar el "git add" .)

$git status (Para ver el status de la base de datos de git.)

$git show (Muestra todos los cambios historicos que has hecho ya sean de html,txt, xml, etc.)

$git show-branch (Muestra todos los cambios historicos en las branch.)

$git show-branch --all(Muestra todos los cambios historicos en las branch pero mas explicado.)

$git log (Para ver el historal de los commits hechos.)

$git log --stat (Para ver el historal de los commits hechos mas específico .)

$git log --graph (Para ver el historal de los commits con mas informacion visual .)

$git push repo1 repo2(Para enviar archivos a otros repositorios,branchs, commits etc. por ejemplo de git a github. )

$git pull repo1 repo2(Para traer archivos de otros repositorios.)

$git config (Para checar las configuraciones de git.)

$git config --list (Para checar las configuraciones de como esta configurado nuestro git)

$git config -global user.name (Para configurar el user local.)

$git config -global user.email (Para configurar el email local.)

$git rm (Para remover archivos que esten en el staging "cache".)

$git diff(Para ver las diferencias de los commits, branches,etc.)

$git checkout (Para checar diferentes commits, branches, etc. Aqui podemos regresarnos a "versiones" antiguas de nuestros archivos.)

$git branch "name del branch" (Para crear branches)

$git merge #main #branch2 (Con merge podemos combinar branches, de las cuales la que sera modificada sera la que este primero y la otra quedara igual.)

$git tag "nombre del tag" (Para darle un nombre a los commit )

$gitk (Muestra visualmente las ramas y tu repositorio, esto no es tan profesional.)

git log --oneline - Te muestra el id commit y el título del commit.

git log --decorate- Te muestra donde se encuentra el head point en el log.

git log --stat - Explica el número de líneas que se cambiaron brevemente.

git log -p- Explica el número de líneas que se cambiaron y te muestra que se cambió en el contenido.

git shortlog - Indica que commits ha realizado un usuario, mostrando el usuario y el titulo de sus commits.

git log --graph --oneline --decorate y

git log --pretty=format:"%cn hizo un commit %h el dia %cd" - Muestra mensajes personalizados de los commits.

git log -3 - Limitamos el número de commits.

git log --after=“2018-1-2” ,

git log --after=“today” y

git log --after=“2018-1-2” --before=“today” - Commits para localizar por fechas.

git log --author=“Name Author” - Commits realizados por autor que cumplan exactamente con el nombre.

git log --grep=“INVIE” - Busca los commits que cumplan tal cual está escrito entre las comillas.

git log --grep=“INVIE” –i- Busca los commits que cumplan sin importar mayúsculas o minúsculas.

git log – index.html- Busca los commits en un archivo en específico.

git log -S “Por contenido”- Buscar los commits con el contenido dentro del archivo.

git log > log.txt - guardar los logs en un archivo txt
